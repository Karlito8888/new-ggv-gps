# ESLint & Prettier Configuration Guide

## üìã Configuration Summary

This project uses **ESLint** for code quality and **Prettier** for code formatting.

### Tools Installed

- **ESLint 9.39.2** - Code quality and best practices
- **Prettier 3.8.1** - Code formatter
- **ESLint Plugins**:
  - `@eslint/js` - Base JavaScript rules
  - `eslint-plugin-react-hooks` - React Hooks validation
  - `eslint-plugin-react-refresh` - React Refresh support

### Configuration Files

| File               | Purpose                                       |
| ------------------ | --------------------------------------------- |
| `eslint.config.js` | ESLint configuration (new flat config format) |
| `.prettierrc.json` | Prettier configuration                        |
| `.prettierignore`  | Prettier ignore patterns                      |

## üöÄ Available Commands

### Linting

```bash
# Check for code quality issues
bun run lint

# Auto-fix ESLint issues
bun run lint:fix
```

### Formatting

```bash
# Format all files with Prettier
bun run format

# Check if files are formatted correctly
bun run format:check
```

### Build & Deploy

```bash
# Build with linting (used by Netlify)
bun run build:netlify
# Runs: eslint . && vite build
```

## ‚öôÔ∏è ESLint Rules

### Enabled Rules

| Rule                                   | Severity | Purpose                                       |
| -------------------------------------- | -------- | --------------------------------------------- |
| `no-unused-vars`                       | error    | Detects unused variables                      |
| `no-console`                           | warn     | Warns on console.log (allows warn/error/info) |
| `no-debugger`                          | warn     | Warns on debugger statements                  |
| `prefer-const`                         | error    | Requires `const` for non-reassigned variables |
| `no-var`                               | error    | Forbids `var` keyword                         |
| `eqeqeq`                               | error    | Requires `===` instead of `==`                |
| `no-eval`                              | error    | Forbids eval() calls                          |
| `prefer-template`                      | warn     | Prefers template literals                     |
| `react-refresh/only-export-components` | warn     | Warns on component export patterns            |

### Disabled Rules (Handled by Prettier)

The following ESLint rules are disabled to avoid conflicts with Prettier:

- `indent`, `quotes`, `semi`, `comma-dangle`
- `object-curly-spacing`, `keyword-spacing`, `space-before-blocks`

## üìù Pre-Commit Workflow

1. **Write Code** - Make your changes
2. **Format** - Run `bun run format` to auto-fix formatting
3. **Lint** - Run `bun run lint` to check quality
4. **Auto-fix** - Run `bun run lint:fix` if needed
5. **Commit** - Only commit after all checks pass

## üîß IDE Integration

### VS Code

The project includes `.vscode/settings.json` with recommended settings. To format on save:

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[javascriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

Install the Prettier VSCode extension:

- **Prettier** - Code formatter by Prettier (esbenp.prettier-vscode)

### WebStorm / IntelliJ

1. Go to **Settings** ‚Üí **Languages & Frameworks** ‚Üí **JavaScript** ‚Üí **Prettier**
2. Enable Prettier and set it as the default formatter
3. Check "Run on save"

## ‚ö° Git Hooks (Optional)

To enforce linting before commits, you can set up `husky`:

```bash
npm install husky lint-staged --save-dev
npx husky install
```

Then create `.husky/pre-commit`:

```bash
#!/bin/sh
. "$(dirname "$0")/_/husky.sh"
npx lint-staged
```

And add to `package.json`:

```json
{
  "lint-staged": {
    "*.{js,jsx}": "eslint --fix",
    "*": "prettier --write"
  }
}
```

## üêõ Debugging ESLint Issues

### Check which rule is triggered

```bash
# Show rule names in linting output
eslint . --format=json | jq '.[] | .messages[] | .ruleId'
```

### Disable a rule for a file

Add at the top of the file:

```js
/* eslint-disable rule-name */
```

### Disable a rule for a single line

```js
// eslint-disable-next-line rule-name
const x = 1;
```
